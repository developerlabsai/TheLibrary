Product Vision & Positioning

A multi-provider waterfall enrichment engine accessible via Slack and Chrome Extension. Upload a file or click a button — get enriched contacts back in minutes. The enrichment tool is intentionally ephemeral: data is processed, delivered to a CRM, and purged. This creates a natural conversion funnel into StackGTM.

The One-Sentence Pitch
"Enrich any sales list in Slack or your browser in 3 minutes. When you're ready to work those leads, send them to your CRM with one click."

What We Are

  • A processing pipeline, not a database — 7-30 day data retention, then purge
  • A waterfall enrichment engine across 5 providers (Apollo, BuiltWith, Wiza, AI Ark, Findymail)
  • A zero-friction UX — upload a CSV in Slack, get enriched contacts back in the same thread
  • A customer acquisition funnel for StackGTM, our CRM

What We Are NOT

  • NOT a CRM (that's StackGTM)
  • NOT a long-term data store — data expires and must be sent to a CRM
  • NOT competing with Clay's 75+ providers — we compete on simplicity
  • NOT a platform for campaigns, sequences, or dialing — those are StackGTM features

The Enrichment-to-CRM Funnel

The enrichment tool is a customer acquisition engine for StackGTM. It intentionally creates a problem it doesn't solve — "I have enriched data but nowhere to manage it." That's not a bug. That's the funnel.

1
Awareness — Chrome Extension on LinkedIn, Slack App Directory, ProductHunt, SEO free tools, referrals
2
Activation — First enrichment delivers instant value (10 seconds for extension, 3 minutes for bulk Slack)
3
Retention Problem — After 3+ enrichments: CSVs scattered everywhere. No central view. No dedup. No follow-up.
4
The Prompt — "Send results to: [CSV] [HubSpot] [StackGTM] [Email]" — every enrichment pushes toward CRM
5
Expansion — Team sees enriched data in StackGTM → installs extension → more enrichments → higher CRM tier

Data Expiration — The Conversion Lever

This is honest and earned urgency. Users are told upfront: this is a processing tool, not storage.

Chrome Extension Architecture

The Chrome Extension is our acquisition engine. While the Slack bot handles bulk enrichment, the extension handles single-contact and small-batch enrichment directly from LinkedIn, company websites, and Google Sheets.

Supported Surfaces

SurfaceDetectionUser Sees
LinkedIn profileCompany + name from DOMEmail, phone, persona, tech stack in sidebar
LinkedIn Sales NavBulk-select visible contacts"Enrich 25 contacts" button overlay
Company websiteDomain detectionTech stack + decision makers in popup
Google SheetsSheet with company columnSidebar: "Enrich this sheet"
HubSpot recordContact/company page"Fill gaps" button on record
Gmail CSV attachmentAttachment detection"Enrich this file" notification

Technical Architecture

Chrome Extension (Manifest V3) content-scripts/ LinkedIn, HubSpot, Sheets, generic popup/ Credit balance, recent enrichments sidepanel/ Enrichment results display (React) background/ API calls, auth token management shared/ API client, types | POST /api/v2/enrich (same backend, new frontend) | Existing waterfall engine (Apollo -> Wiza -> AI Ark) | Results -> Extension sidebar / Slack / Email / CRM

Key Differentiator vs. Lusha/Apollo

Lusha and Apollo extensions query one provider. Ours runs a waterfall across multiple providers and returns the best composite result. Apollo found the email but no phone? Wiza found the phone. AI Ark verified both. One click.

Auth Flow

Extension → Opens landing page → Slack OAuth (existing identity layer) → Token stored in chrome.storage → All API calls authenticated. No new auth system needed.

Build Estimate
2-3 weeks of focused work. The enrichment API exists, React components exist (admin dashboard), Slack OAuth exists. The extension is a frontend shell calling existing endpoints.

The 5 Viral Loops

No single loop achieves 10/10 virality. Five loops at 6-7/10 each compound into an 8/10 system. These run simultaneously and reinforce each other.

Loop 1

Free Public Tools (SEO + Share Magnet)

Ungated micro-tools that generate shareable pages and SEO traffic. Company tech stack lookup, free email verifier, "Who Uses [Technology]?" explorer. Every shared link is a branded URL back to us.

Viral MomentSales reps share tech stack pages in Slack groups, LinkedIn posts, and cold emails. Every share is organic brand exposure.
Loop 2

Shareable Enrichment Reports

Every enrichment generates a branded interactive report page instead of just a CSV. Manager says "forward me that list" — rep shares a link. Manager sees a beautiful branded report.

Viral MomentReports shared internally spread the brand to decision-makers who authorize team-wide purchases.
Loop 3

Chrome Extension Visual Footprint

The extension injects a branded sidebar visible on LinkedIn. Visible in screen shares, screenshots, standups. "Share this contact" button generates branded card images.

Viral Moment"What tool is that?" — asked in every sales team meeting when the sidebar appears during screen shares.
Loop 4

Credit Referral Engine

Share link → referee completes first enrichment → both get 50 credits. No cap. Contextual prompt after every job: "This cost 23 credits. Share with a colleague and you'll both get 50 free."

Viral Moment"I got $200 in free enrichment credits this month from referrals" — a LinkedIn post they write themselves.
Loop 5

"Powered by" Data Watermark

Every piece of data that leaves the system carries the brand. CSV exports include enrichment_source = StackGTM column. CRM syncs set the same property. Email delivery includes branded footer.

Viral MomentSales ops sees 500 contacts in HubSpot with "StackGTM" source → Googles it → signs up.

Open Source Strategy

License: AGPLv3

Forces competitors to open-source their modifications. Allows self-hosting for personal/small team use. Commercial use requires a license. This is the n8n playbook: OSS core → managed cloud → enterprise licenses.

What to Open Source

  • Slack bot + file upload + AI intent classification
  • BuiltWith, Apollo, Wiza, AI Ark API clients
  • Waterfall enrichment orchestration engine
  • BullMQ job queue + workers
  • Prisma schema + migrations
  • Docker setup + deployment guides

What Stays Proprietary (Cloud Only)

  • Admin dashboard & analytics
  • Chrome Extension
  • CRM integrations (HubSpot, StackGTM)
  • Billing & credit system
  • Multi-tenant licensing
  • Free public tools (SEO engine)
  • Referral system
  • Shareable reports
Community Benefit
Developer community contributes provider integrations. You don't need to build every enrichment provider — the community does it. GitHub stars → HackerNews → ProductHunt → viral loop.

Why We Win

  • Multi-provider waterfall: No other extension or Slack app does this. We composite the best data from 5 providers.
  • BYOK model: Zero-cost entry. No competitor offers this.
  • Slack-native: Teams already live in Slack. No context switching.
  • Vertical integration: We own both enrichment AND the CRM (StackGTM). Clearbit couldn't do this.
  • Ephemeral by design: Privacy-first, GDPR-friendly. No liability of storing millions of contacts.

The 10/10 Virality Gap
True 10/10 virality requires the product's output to be sent to non-users with forced adoption (like Calendly links or DocuSign requests). Enriched CSVs are consumed privately. Our 5-loop system achieves 8/10 — growing fast with compounding loops. The gap to 10/10 closes in Year 2-3 with platform dynamics (developer marketplace, API ecosystem).
